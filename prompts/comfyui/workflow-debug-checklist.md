# ComfyUI 工作流报错排查提示词

## 资产信息

| 字段 | 内容 |
|------|------|
| 类型 | ComfyUI / 排错 |
| 适用场景 | 工作流导入后跑不起来，需要整理排查步骤 |
| 适用模型/平台 | ChatGPT / Codex |
| 输入变量 | `{报错信息}`、`{工作流来源}`、`{已安装模型}`、`{系统环境}` |
| 输出结果 | 排查清单和下一步操作 |
| 标签 | ComfyUI, 报错, 工作流, 新手 |
| 版本 | v1.0 |

## 标准版提示词

```text
请帮我分析这个 ComfyUI 工作流为什么跑不起来。

报错信息：
{报错信息}

工作流来源：
{工作流来源}

我已经安装的模型或节点：
{已安装模型}

系统环境：
{系统环境}

请按新手能理解的方式输出：
1. 报错大概率属于哪一类
2. 先检查哪些文件或目录
3. 可能缺哪些模型、节点或插件
4. 不建议新手乱改的地方
5. 下一步最小排查动作
```

## 增强版提示词

```text
你是一名 ComfyUI 工作流排错助手。请根据用户提供的报错信息和环境，生成一个从低风险到高风险的排查方案。

要求：
- 不要让用户一上来重装环境
- 先检查模型路径、缺失节点、插件依赖、文件名、显存和输入输出连接
- 每一步说明“为什么检查”和“如何判断是否通过”
- 如果信息不足，列出需要补充的信息
- 输出适合新手照着做的清单

输入：
报错信息：{报错信息}
工作流来源：{工作流来源}
已安装模型/节点：{已安装模型}
系统环境：{系统环境}
```

## 使用建议

适合公众号、知识星球答疑和 B站评论区常见问题整理。正式排错时最好附上完整报错截图和工作流来源。
