# RunningHub 工作流转本地检查提示词

## 资产信息

| 字段 | 内容 |
|------|------|
| 类型 | ComfyUI / RunningHub |
| 适用场景 | 将云端工作流迁移到本地前检查差异 |
| 适用模型/平台 | ChatGPT / Codex |
| 输入变量 | `{云端工作流说明}`、`{本地环境}`、`{模型清单}`、`{插件清单}` |
| 输出结果 | 迁移检查表、风险点、准备清单 |
| 标签 | RunningHub, ComfyUI, 工作流迁移, 云端 |
| 版本 | v1.0 |

## 标准版提示词

```text
请帮我判断这个 RunningHub / 云端 ComfyUI 工作流能不能迁移到本地运行。

云端工作流说明：{云端工作流说明}
本地环境：{本地环境}
本地已有模型：{模型清单}
本地已有插件：{插件清单}

请输出：
1. 迁移难度
2. 需要准备的模型
3. 需要安装的自定义节点
4. 显存和性能风险
5. 哪些步骤可以继续在云端跑
6. 本地迁移最小行动清单
```

## 增强版提示词

```text
你是 ComfyUI 云端到本地迁移顾问。请把用户的云端工作流拆成模型依赖、节点依赖、硬件依赖和操作风险。

要求：
- 先判断是否值得迁移
- 如果本地成本过高，建议继续云端运行
- 不要默认用户有高端显卡
- 给出低风险迁移顺序
- 标注哪些文件需要用户自行下载或授权

输出：
## 是否建议迁移
## 依赖清单
## 本地风险
## 云端继续运行的理由
## 迁移步骤
## 新手注意事项
```

## 使用建议

适合承接 RunningHub 轻植入：先让新手在云端跑通，再解释什么时候值得转本地。
