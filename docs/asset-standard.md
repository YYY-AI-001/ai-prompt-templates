# 提示词资产标准

## 核心定义

提示词资产不是一段孤立 prompt，而是一套可复用说明。

一个合格资产至少回答：

- 用来做什么？
- 适合哪个平台或模型？
- 哪些部分可以替换？
- 输出应该长什么样？
- 使用时容易踩什么坑？

## 推荐字段

| 字段 | 说明 |
|------|------|
| 标题 | 用一句话说明资产用途 |
| 类型 | 内容创作 / AI 生图 / AI 视频 / ComfyUI / 其他 |
| 适用场景 | 真实使用场景 |
| 适用模型/平台 | ChatGPT、FLUX、Midjourney、Seedance、ComfyUI 等 |
| 输入变量 | 可替换字段，如 `{主题}`、`{目标用户}` |
| 标准版提示词 | 日常可复制版本 |
| 增强版提示词 | 更强约束、更专业版本 |
| 输出格式 | 期望模型返回的结构 |
| 使用建议 | 什么时候用，怎么改 |
| 标签 | 方便检索 |
| 版本记录 | 记录迭代 |

## 命名规范

文件名使用小写英文和连字符：

```text
bilibili-cover.md
seedance-shot-prompt.md
workflow-debug-checklist.md
```

## 变量规范

变量使用中文花括号：

```text
{主题}
{目标用户}
{风格}
{参考图描述}
{输出格式}
{限制条件}
```

## 质量标准

- 目标明确
- 变量清楚
- 输出格式明确
- 约束不冲突
- 适合真实任务
- 能被下次复用
