# ComfyUI 模型路径检查清单

## 资产信息

| 字段 | 内容 |
|------|------|
| 类型 | ComfyUI / 排错 |
| 适用场景 | 检查模型、LoRA、VAE、ControlNet 等文件是否放对位置 |
| 适用模型/平台 | ChatGPT / Codex |
| 输入变量 | `{报错信息}`、`{模型文件名}`、`{ComfyUI目录}`、`{工作流要求}` |
| 输出结果 | 路径检查清单、缺失文件判断 |
| 标签 | ComfyUI, 模型路径, 报错, 新手 |
| 版本 | v1.0 |

## 标准版提示词

```text
请帮我检查 ComfyUI 模型路径问题。

报错信息：{报错信息}
模型文件名：{模型文件名}
ComfyUI 目录：{ComfyUI目录}
工作流要求：{工作流要求}

请输出：
1. 可能缺失的模型类型
2. 每类模型应该放在哪个目录
3. 文件名是否可能不匹配
4. 是否需要刷新 ComfyUI 或重启
5. 下一步最小排查动作
```

## 增强版提示词

```text
你是 ComfyUI 模型路径排查助手。请根据报错和工作流要求，判断用户缺的是 checkpoint、vae、clip、lora、controlnet、upscale model 还是自定义节点。

输出格式：
## 报错归类
## 可能缺失文件
## 推荐检查目录
## 文件名匹配风险
## 插件依赖风险
## 低风险修复步骤
## 仍然失败时需要补充的信息
```

## 使用建议

适合处理“工作流导入后红框、找不到模型、节点报错”的新手问题。不要一上来建议重装环境。
